Interactions Endpoint (Discord) — Page de test

⚠️ Cette page n’est pas l’endpoint “Interactions” pour Discord. Discord exige une URL serveur qui reçoit des POST et vérifie la signature. Sur Netlify, ça se fait via une Function.

URL à mettre dans Discord

Dans le Developer Portal, le champ URL du point de terminaison des interactions doit viser ton endpoint serveur, par exemple : 


    

      Exemple Netlify Function : https://TON-SITE.netlify.app/.netlify/functions/interactions

      (ou si tu as un redirect : https://TON-SITE.netlify.app/api/interactions → function)

Rappel : pourquoi un HTML ne suffit pas ?

    
    
      
  • Discord appelle ton endpoint depuis ses serveurs (pas depuis ton navigateur).
    • 
      
  • Il faut vérifier la signature Ed25519 → ça doit être fait côté serveur (Function).
    • 
      
  • Une page HTML ne peut pas “recevoir” les POST Discord comme un serveur.
